# Мета роботи: Ознайомитись з ключовими поняттями об’єктно-орієнтованого програмування (ООП) у Python та навчитися реалізовувати їх у власних класах на прикладі ігрової симуляції.

Виконання роботи
# 1. Інкапсуляція
У ході виконання завдання було реалізовано клас BankAccount з приватним атрибутом __balance.
• Результат: Додано цикл, що 5 разів генерує випадкову суму та тип операції (поповнення або зняття).
• Висновок: Ми обмежили прямий доступ до балансу, забезпечивши безпеку даних.

# 2. Наслідування
Створено базовий клас Vehicle та дочірній Car.
• Результат: Додано метод service_info() у батьківський клас, який успішно викликається через об'єкт класу Car.
• Висновок: Навчилися передавати функціонал від загальних класів до конкретних.

# 3. Поліморфізм
Реалізовано перевірку поведінки об'єктів при відсутності перевизначеного методу.
• Результат: Клас Fish не має власного методу speak(), тому при виклику спрацьовує логіка базового класу Animal.
• Висновок: Поліморфізм дозволяє використовувати однаковий інтерфейс для різних типів об'єктів.

# 4. Абстракція та ігрова симуляція (Індивідуальне завдання)
Створено ієрархію бойових предметів на основі абстрактного класу Item.
• Розробили: Клас Bow (Лук) з параметром range_power та методами attack() і reload().
• Результат: Реалізовано покрокову гру, де гравець обирає дію (атака або посилення), а ворог обирається випадковим чином.
